# Free vs Paid Boundary

## Free/open-source

The repo should generously publish primitives that help people adapt:

- Markdown templates
- JSON schemas
- Local CLI validators
- Browser-local demos
- Example packets
- Printable worksheets
- Public-safe educational explanations

Free tools should be useful without requiring an account, tracking, a subscription, or private data upload.

## Paid later

Paid products should add real value beyond what a free AI prompt or static worksheet can provide:

- Hosted private dashboard
- Team/institution customization
- Review by Hypernovelty/iPublishOS operators
- Sector-specific adaptation reports
- Workshops and briefings
- Workflow implementation support
- Evidence/provenance QA packages
- Ongoing signal monitoring and update cadence

## Never fake scarcity

Do not hide basic adaptation knowledge behind a paywall. Paid products should charge for depth, implementation, review, integration, or support — not for withholding the core concepts.
