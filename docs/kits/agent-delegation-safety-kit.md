# Agent Delegation Safety Kit

**Public promise:** Before an AI agent touches tools, files, customers, accounts, money, publishing, or internal systems, make its authority reviewable.

**Audience:** Team leads, ops owners, security/IT partners, managers, creators using agent workflows.

## Use this when

- A person or team has a concrete decision to make under fast-changing AI/information conditions.
- The output needs to be reviewable without uploading private data.
- A synthetic example is enough to teach the workflow before customization.

## Included primitives

- AI Delegation Contract Card
- Agent Permission Receipt Card
- Agent Identity & Scope Roster
- Non-Human Identity Review Receipt
- Agent Revocation Reality Check

## Candidate extension handling

Agent Onboarding Packet Lite should be a bundle/index card that references these existing tools, not primitive #31.

## Boundary

Not a security certification, legal/compliance clearance, or deployment approval.

## Open-source release notes

- Keep examples synthetic.
- Keep browser-local/offline paths prominent.
- Do not imply legal, security, academic-integrity, medical, financial, or compliance authority.
- Paid value later must come from customization, facilitation, hosting, recurring review, or implementation support — not from guaranteed outcomes.
