# Shadow AI Amnesty Triage Sheet: Student AI debugging habit to curriculum evidence

**Review owner:** Instructor or course assessment lead
**Review date:** 2026-07-09

## Context
Synthetic statistics course example: a student used a personal AI chat to debug homework code because office hours were full and the course policy did not explain what AI debugging help could be disclosed or how concept understanding would be checked.

## Job to be done
Translate a hidden AI debugging habit into an allowed support boundary, a disclosure rule, and concept-check evidence that shows the student understands the statistical method and code behavior.

## Why approved tools failed
- Policy said broad AI use was risky but did not distinguish syntax help, debugging hints, full solution generation, or concept explanation.
- Students had no low-friction way to disclose debugging help without fearing automatic punishment.
- The assignment graded final code more clearly than reasoning, error diagnosis, or interpretation of results.

## Data touched
- Synthetic homework prompt and toy dataset only
- Student-owned draft code with no classmates, grades, health, financial, account, or private records
- AI transcript excerpt summarized by the student rather than pasted into a public repo

## Risk exposure
- AI could have generated the substantive solution instead of a debugging hint
- Student may over-rely on AI explanations without detecting wrong statistical reasoning
- Undisclosed use makes the instructor unable to repair the assignment design
- Uploading real student/class data would be a separate policy and privacy issue

## Safer workaround
- Allow AI for syntax/debugging hints only when the student discloses the kind of help received
- Require a short concept check: explain the model choice, the bug, the fix, and one result interpretation in the student's own words
- Require the student to mark which lines were AI-suggested or AI-debugged when help shaped the final code
- Prohibit uploading private course data, answer keys, or another student's work
- Offer an instructor-reviewed example of acceptable debugging disclosure

## Policy/product owner
Course instructor + program assessment lead + local academic-integrity policy owner

## Non-punitive next step
Use this pattern to update the assignment instructions: separate allowed debugging support from prohibited solution outsourcing, add a disclosure line, and add a concept-check evidence requirement before escalating ambiguous cases.

## Do not claim
- This is not permission to cheat or outsource the assignment
- This is not proof that AI-generated code is correct
- This is not a universal academic-integrity policy
- This is not approval to upload private student/course data to public AI tools

## Boundary note
Education example only. Local instructor and institutional policy review remain required. The goal is to convert shadow AI habits into teachable boundaries and learning evidence, not to normalize secrecy, data leakage, or detector-first punishment.
