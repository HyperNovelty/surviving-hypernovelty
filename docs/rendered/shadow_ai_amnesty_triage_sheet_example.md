# Shadow AI Amnesty Triage Sheet: Nonprofit comment-summary triage

**Review owner:** Program operations manager
**Review date:** 2026-07-06

## Context
A program coordinator used a personal AI tool to summarize non-sensitive public workshop comments because the approved survey tool lacked bulk summarization.

## Job to be done
Summarize recurring themes from workshop feedback before a staff planning meeting.

## Why approved tools failed
- Approved survey export was hard to read quickly
- No internal summarization workflow existed
- Deadline pressure pushed staff toward an available personal tool

## Data touched
- Public workshop comments with names removed
- No health, legal, financial, student, client, or account records in this synthetic example

## Risk exposure
- Staff may repeat this pattern with more sensitive data later
- No shared record of what was pasted or what the tool retained
- Policy looked like a ban but did not offer a workable alternative

## Safer workaround
- Use redacted CSV with no names/contact details
- Summarize locally or with approved workspace tool only
- Keep a one-line source/use log
- Escalate if comments include private or sensitive details

## Policy/product owner
Operations lead + IT/tool owner

## Non-punitive next step
Treat this as a workflow gap: define a safe comment-summary path and update the micro-policy before warning staff.

## Do not claim
- This is not permission to upload confidential data
- This is not a compliance clearance
- This is not an employee discipline record

## Boundary note
Local non-punitive triage aid only. It should surface unmet workflow demand and safer options, not encourage evasion or normalize private-data uploads.
