## What this supports
- The file appears to have an inspectable origin/edit history in this synthetic example
- The credential may help separate original capture from later screenshot/repost copies

## What this does not prove
- It does not prove the caption is true
- It does not prove consent, fairness, or newsworthiness
- It does not prove the image was taken at the claimed event

## Context checks
- Check event date/location against primary organizer page
- Ask source how they obtained the file
- Do not reuse a screenshot as if credentials survived
- Verify caption separately from image origin
